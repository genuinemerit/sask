### Issues encountered and resolved

| Issue | Root cause | Fix |
|-------|-----------|-----|
| SSH unreachable | `~` not shell-expanded in `ansible.cfg` `ssh_args = -F ~/.ssh/...` | Removed `-F`; main `~/.ssh/config` includes `config.d/*` |
| venv creation failed | `pip` module received controller-local path instead of remote path | Copy `requirements.txt` to droplet first; pip installs from remote path |
| Caddy reload failed (permission denied on `sask.log`) | Template `validate:` ran `caddy` as root, creating log file owned root:root | Removed validate; recurse chown of `/var/log/caddy` placed after Caddyfile deploy |
| Caddyfile format warning | Spaces instead of tabs | Fixed template indentation to tabs |
| Non-idempotent `requirements.txt` copy | Copied as `root:root`, recurse task changed to `sask:sask`; conflict on re-run | Fixed copy owner to `sask:sask` |
| Non-idempotent `src/sask` sync | `copy` module re-copied `__pycache__` (pyc content varies) and rsync `-o` flag reset ownership each run | Switched to `ansible.posix.synchronize` with `--exclude=__pycache__ --no-owner --no-group` |
